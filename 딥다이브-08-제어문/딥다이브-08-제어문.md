# [JavaScript] 제어문

(JavaScript Deep Dive 93쪽)

## 블록문

---

- 블록문은 0개 이상의 문을 {중괄호}로 묶은 것으로, 코드 블록 또는 블록이라고 함
- 자바스크립트에서 블록문은 하나의 실행 단위로 취급
- 블록문은 자체종결성을 갖기 때문에 세미콜론(;)을 붙이지 않음

## 조건문

---

- 조건문은 주어진 조건식의 평가 결과에 따라 코드 블록(블록문)의 실행을 결정함

### `if...else 문`

- `if...else문` 은 주어진 조건식이 true 면 if 코드블록이, false 일 경우 Else 코드블록이 실행됨
    
    ```jsx
    if (조건식) {
    	// 조건식이 참이면 실행
    } else {
    	// 조건식이 거짓이면 실행
    }
    ```
    
- 대부분의 `if...else`문은 삼항조건연산자로 바꿔쓸 수 있음
    
    ```jsx
    var num = 2
    
    var kind = num ? (num > 0 ? '양수' : '음수') : '영'
    
    console.log(num) // 양수
    ```
    

### `switch 문`

- switch 문은 주어진 표현식을 평가하여 그 값과 일치하는 표현식을 갖는 case문으로 실행 흐름을 옮김
- switch 문의 표현식과 일치하는 case 문이 없다면 실행 순서는 default 문으로 이동함
    
    ```jsx
    switch (표현식) {
    	case 표현식1:
    		switch 문의 표현식과 표현1이 일치하면 실행될 문
    		break
    	case 표현식2
    		switch 문의 표현식과 표현식2가 일치하면 실행될 문
    		break
    	default
    		switch 문의 표현식과 일치하는 case문이 없을 경우 실행될 문
    }
    ```
    
    주의할점: break 문을 매 case 마다 쓰지 않으면 의도치않게 마지막 `default` 문까지 실행될 수 있음
    
    의도한 바가 아니면 `break` 를 써주는 것이 좋다
    

## 반복문

---

반복문은 조건식의 평가 결과가 참인 경우 코드 블록을 실행하고,
그후 조건식을 다시 평가하여 조건식이 거짓일 때까지 반복됨

### `for 문`

```jsx
for (let i = 0; i < array.length; i++) {
	...
	console.log(i)
}
```

### `while 문`

- for 문은 반복 횟수가 명확할 때 사용하고, while 문은 반복 횟수가 불분명할 때 사용한다
    
    ```jsx
    var count = 0
    
    while (count < 3) {
    	console.log(count) // 0 1 2
    	count++
    }
    ```
    

### `do...while 문`

- do...while문은 코드 블록을 먼저 실행하고 조건식을 평가함
- 따라서 코드 블록은 무조건 한번 이상 실행됨
    
    ```jsx
    var count = 0
    
    do {
    	console.log(count) // 0 1 2
    	count++
    } while (count < 3)
    ```
    

## `break 문`

---

- switch문과 while문에서 `break` 문은 코드 블록을 탈출함
- 또한 식별자가 붙은 `레이블 문` 을 탈출함
    
    ```jsx
    // foo라는 식별자가 붙은 레이블 블록문
    foo: {
    	console.log(1)
    	break foo // foo 레이블 블록문을 탈출함
    	console.log(2)
    }
    
    console.log('Done!')
    ```
    
- 중첩된 for 문의 내부 for 문에서 break 문을 실행하면 내부 for문을 탈출하여 외부 for문으로 진입하다
(for문 전체가 아예 종료되는 게 아니라는 말씀)
    
    ```jsx
    // outer라는 식별자가 붙은 레이블 for문
    outer: for (var i = 0; i < 3; i++) {
    	for (var j = 0; j < 3; j++) {
    		// i + j === 3 이면 Outer 라는 식별자가 붙은 레이블 for문을 탈출한다
    		if (i + j ===3) break outer;
    		console.log(`inner [${i}, ${j}]`)
    	}
    }
    ```
    
    > 레이블 문은 중첩된 for 문 외부로 탈출할 때 유용하지만 그 밖의 경우에는 일반적으로 권장하지 X
    > 
    > 
    > 코드 가독성이 나빠지고, 오류 가능성 때문
    > 

## `continue 문`

---

- continue문은 반복문의 코드 블록 실행을 현 지점에서 중단하고 반복문의 증감식으로 실행 흐름을 이동시킴
- break문처럼 반복문을 탈출하지는 않음
    
    ```jsx
    var string = 'Hello world.'
    var search = 'l'
    var count = 0
    
    // 문자열은 유사 배열이므로 for 문으로 순회할 수 있다.
    for (var i = 0; i < string.length; i++) {
    	// 'l'이 아니면 현 지점에서 실행을 중단하고 반복문의 증감식으로 이동한다
    	if (string[i] !== search) continue
    	count++
    }
    
    console.log(count) // 3 - 'l'이 3번 걸렸기 때문
    ```
