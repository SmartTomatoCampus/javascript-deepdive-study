## 함수란?

입력을 받아 출력을 내보내는 일련의 과정을 정의한 것

**함수는 값이며, 식별자인 함수 이름을 통해 구분**한다

<aside>
📌 **함수는 객체다!**
하지만 일반 객체와는 다르다. 일반 객체는 호출할 수 없지만 **함수는 호출할 수 있다**
그리고 **함수 객체만의 고유한 프로퍼티**를 갖는다

</aside>

### 함수 정의

```jsx
function funcName (x, y ) {
	...
	return x+y;
}
```

### 함수 호출

```jsx
function funcName (x, y ) {
	...
	return x+y;
}

funcName(1, 2); //함수 이름으로 호출
```

### 함수 동작 순서

선언 ⇒ 호출 ⇒ 실행 ⇒ 결과값 리턴

### 함수를 사용하는 이유

같은 코드를 몇 번이고 재사용할 수 있어, 유지보수의 편의성이 높고 코드의 신뢰성을 높일 수 있다

식별자로 구분할 수 있기 때문에 코드의 가독성을 향상시킨다

## 함수 리터럴

function 키워드, 함수 이름, 매개변수 목록, 함수 몸체로 구성

```jsx
//변수에 함수 리터럴을 할당
var f = function add(x, y){
	return x + y;
};
```

| 구성요소 | 설명 |
| --- | --- |
| 함수 이름 | 1. 식별자 네이밍 규칙 준수
2. 함수 이름은 함수 몸체 내에서만 참조할 수 있는 식별자
3. 이름 생략 가능 ( 이름 없는 함수를 익명함수라고 한다) |
| 매개변수 목록 | 1. 0개 이상의 매개변수를 소괄호로 감싸고 쉼표로 구분
2. 함수 호출할 때 지정한 인수가 순서대로 할당
3. 함수 몸체 내에서 변수와 동일하게 취급되며, 식별자 네이밍 규칙을 준수해야한다’ |
| 함수 몸체 | 1. 함수가 호출되었을 때 실행될 문들을 하나의 실행 단위로 정의한 것
2. 함수 몸체는 함수 호출에 의해 실행됨 |

## 함수 정의

함수를 호출하기 매개변수, 실행할 문, 반환 값을 지정하는 것을 말한다

정의된 함수는 자바스크립트 엔진에 의해 평가되어 함수 객체가 된다

**함수 정의하는 방법**

| 함수 정의 방식 | 예시 |
| --- | --- |
| 함수 선언문 | function add ( x, y) {
   return x + y ;
} |
| 함수 표현식 | var add = function add ( x, y ) {
   return x + y ; 
} |
| Function 생성자 함수 | var add = new Function( ‘x’ + ‘y’, ‘return x + y’); |
| 화살표 함수 | var add = (x , y) ⇒ x + y ; |

### 함수 선언문

함수 선언문은 함수 이름을 생략할 수 없다

<aside>
📌 **함수는 표현식이 아닌 문이다! ⇒ 그래서 선언문이라고 부름
표현식이 아닌 문은 변수에 할당될 수 없다** 하지만

```jsx
var add = function add ( x, y) {
   return x + y ; 
}
```

이 코드를 보면 함수 선언문이 변수에 할당 되는 것처럼 보인다

이렇게 동작하는 이유는 자바스크립트 엔진이 코드 문맥에 따라 동일한 함수 리터럴을 표현식이 아닌 선언문으로 해석하거나 리터럴 표현식으로 해석하는 경우가 있기 때문이다
⇒ 기명함수 리터럴은 함수 선언문 또는 함수 리터럴 표현식으로 해석

**{ }은 블록문일 수도 있고 객체 리터럴 일 수 있다**

```jsx
function foo() {
	console.log('foo');
}
foo();

(function bar() { console.log('bar');});
bar(); //ReferenceError : bar is not defined 
```

foo는 함수 선언문으로 해석되지만 bar는 함수 리터럴 표현식으로 해석된다
따라서 선언문은 피연산자 값으로 사용할 수 없다

<aside>
❓ **함수 이름은 함수 몸체내에서만 참조할 수 있는 식별자다? 는 무슨 말이였을까
함수 몸체 외부에서는 함수 이름으로 함수를 참조할 수 없으므로 함수 몸체 외부에서는 함수이름으로 함수를 호출할 수 없다**

</aside>

⇒그래서 bar는 함수를 가리키는 식별자가 없기 때문에 bar 함수는 호출할 수 없다
foo도 원래는!! 호출 할 수 없어야한다 그런데 자바스크립트는 암묵적으로 식별자를 생성하여 함수를 호출할 수 있게 한다

```jsx
var add = function foo() {
	console.log('foo');
}
로 자바스크립트가 암묵적으로 식별자를 생성한것!
```

**자바스크립트 엔진은 생성된 함수를 호출하기 위해 함수 이름과 동일한 이름의 식별자를 암묵적으로 생성하고 함수 객체를 할당한다**

## 결론 : 함수는 함수 객체를 가리기는 식별자로  호출

</aside>

### 함수 표현식

함수는 일급 객체이므로 함수 리터럴로 생성한 함수 객체를 변수에 할당 할 수 있다

```jsx
var add = function foo() {
	console.log('foo');
};
console.log(add(2,3)); //5
console.log(foo(2,3)); // ReferenceError 남
```

### Function 생성자 함수

생성자 함수에 매개변수 목록과 함수 몸체를 문자열로 전달하면 new 연산자와 함께 함수 객체를 생성하여 반환

```jsx
var add = new Function( ‘x’ + ‘y’, ‘return x + y’);
```

근데 이렇게 생성하는 일은 거의 없고 권장하지도 않으니 쓰지말자

클로저를 생성하지 않고 함수 선언문이나 표현식으로 생성한 함수와 다르게 동작한다

```jsx
var add = ( function () {
	var a = 10;
	return function (x , y) {
		return x+y+a;
		};
}()};

console.log(add(1,2)); //13

var add = ( function () {
	var a = 10;
	return  new Function( ‘x’ + ‘y’, ‘return x + y + a;’);
}()};
console.log(add(1,2)); //RefernceError : a is not defined
```

### 화살표함수

function 키워드 대신 화살표 ⇒ 를 이용하여 간랸하게 함수 선언하며 익명 함수로 정의한다

```jsx
var add = (x , y) ⇒ x + y ;
console.log(add(2,5));
```

**기존 함수와 조금 다른 점이 있다.** 

- 생성자 함수로 사용할 수 없음
- 기존 함수랑 this 바인딩 방시이 다름
- prototype 프로퍼티가 없음
- arguments 객체를 생성하지 않음

## 함수 호이스팅

함수 선언문이 코드 선두로 끌어 올려진 것 처럼 동작하는 자바스크립트 고유 특징

함수 선언문으로 정의한 함수는 함수 선언문 이전에 호출할 수 있으나 함수 표현식으로 정의한 함수는 호출할 수 없다

⇒ 함수 선언문과 함수 표현식의 생성 시점이 다르기 때문!
     함수 선언문은 런타임 이전에 실행되어 자바스크립트 엔진이 암묵적으로 식별자를 생성
     함수 표현식은 런타임에 평가 되므로 변수 호이스팅처럼 발생된다

> 하지만 변수 호이스팅과는 미묘한 차이가 있다. 변수는 undefined로 초기화 되고 암묵적 함수 식별자는 함수 객체로 초기화 된다.
> 

**함수 호이스팅은 함수를 호출 하기전에 반드시 함수를 호출해야 한다는 규칙을 무시하므로 함수 표현식 쓸 것을 권장한다**

**함수 표현식과 함수 선언식의 차이**
함수 선언식은 선언문이다. 그래서 자바스크립트가 암묵적으로 함수 이름과 같은 식별자를 생성한다. 
함수 표현식은 변수에 함수 객체를 할당한 것이다. 그래서 호이스팅에 영향받지 않는다 
그래서 선언하지도 않은 함수를 선언해도 실행되어 규칙을 무시하므로 함수 표현식을 쓰자
