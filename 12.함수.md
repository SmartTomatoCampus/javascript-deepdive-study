## 함수란?

입력을 받아 출력을 내보내는 일련의 과정을 정의한 것

**함수는 값이며, 식별자인 함수 이름을 통해 구분**한다

<aside>
📌 **함수는 객체다!**
하지만 일반 객체와는 다르다. 일반 객체는 호출할 수 없지만 **함수는 호출할 수 있다**
그리고 **함수 객체만의 고유한 프로퍼티**를 갖는다

</aside>

### 함수 정의

```jsx
function funcName (x, y ) {
	...
	return x+y;
}
```

### 함수 호출

```jsx
function funcName (x, y ) {
	...
	return x+y;
}

funcName(1, 2); //함수 이름으로 호출
```

### 함수 동작 순서

선언 ⇒ 호출 ⇒ 실행 ⇒ 결과값 리턴

### 함수를 사용하는 이유

같은 코드를 몇 번이고 재사용할 수 있어, 유지보수의 편의성이 높고 코드의 신뢰성을 높일 수 있다

식별자로 구분할 수 있기 때문에 코드의 가독성을 향상시킨다

## 함수 리터럴

function 키워드, 함수 이름, 매개변수 목록, 함수 몸체로 구성

```jsx
//변수에 함수 리터럴을 할당
var f = function add(x, y){
	return x + y;
};
```

| 구성요소 | 설명 |
| --- | --- |
| 함수 이름 | 1. 식별자 네이밍 규칙 준수
2. 함수 이름은 함수 몸체 내에서만 참조할 수 있는 식별자
3. 이름 생략 가능 ( 이름 없는 함수를 익명함수라고 한다) |
| 매개변수 목록 | 1. 0개 이상의 매개변수를 소괄호로 감싸고 쉼표로 구분
2. 함수 호출할 때 지정한 인수가 순서대로 할당
3. 함수 몸체 내에서 변수와 동일하게 취급되며, 식별자 네이밍 규칙을 준수해야한다’ |
| 함수 몸체 | 1. 함수가 호출되었을 때 실행될 문들을 하나의 실행 단위로 정의한 것
2. 함수 몸체는 함수 호출에 의해 실행됨 |

## 함수 정의

함수를 호출하기 매개변수, 실행할 문, 반환 값을 지정하는 것을 말한다

정의된 함수는 자바스크립트 엔진에 의해 평가되어 함수 객체가 된다

**함수 정의하는 방법**

| 함수 정의 방식 | 예시 |
| --- | --- |
| 함수 선언문 | function add ( x, y) {
   return x + y ;
} |
| 함수 표현식 | var add = function add ( x, y ) {
   return x + y ; 
} |
| Function 생성자 함수 | var add = new Function( ‘x’ + ‘y’, ‘return x + y’); |
| 화살표 함수 | var add = (x , y) ⇒ x + y ; |

### 함수 선언문

함수 선언문은 함수 이름을 생략할 수 없다

<aside>
📌 **함수는 표현식이 아닌 문이다! ⇒ 그래서 선언문이라고 부름
표현식이 아닌 문은 변수에 할당될 수 없다** 하지만

```jsx
var add = function add ( x, y) {
   return x + y ; 
}
```

이 코드를 보면 함수 선언문이 변수에 할당 되는 것처럼 보인다

이렇게 동작하는 이유는 자바스크립트 엔진이 코드 문맥에 따라 동일한 함수 리터럴을 표현식이 아닌 선언문으로 해석하거나 리터럴 표현식으로 해석하는 경우가 있기 때문이다
⇒ 기명함수 리터럴은 함수 선언문 또는 함수 리터럴 표현식으로 해석

**{ }은 블록문일 수도 있고 객체 리터럴 일 수 있다**

```jsx
function foo() {
	console.log('foo');
}
foo();

(function bar() { console.log('bar');});
bar(); //ReferenceError : bar is not defined 
```

foo는 함수 선언문으로 해석되지만 bar는 함수 리터럴 표현식으로 해석된다
따라서 선언문은 피연산자 값으로 사용할 수 없다

<aside>
❓ **함수 이름은 함수 몸체내에서만 참조할 수 있는 식별자다? 는 무슨 말이였을까
함수 몸체 외부에서는 함수 이름으로 함수를 참조할 수 없으므로 함수 몸체 외부에서는 함수이름으로 함수를 호출할 수 없다**

</aside>

⇒그래서 bar는 함수를 가리키는 식별자가 없기 때문에 bar 함수는 호출할 수 없다
foo도 원래는!! 호출 할 수 없어야한다 그런데 자바스크립트는 암묵적으로 식별자를 생성하여 함수를 호출할 수 있게 한다

```jsx
var add = function foo() {
	console.log('foo');
}
로 자바스크립트가 암묵적으로 식별자를 생성한것!
```

**자바스크립트 엔진은 생성된 함수를 호출하기 위해 함수 이름과 동일한 이름의 식별자를 암묵적으로 생성하고 함수 객체를 할당한다**

## 결론 : 함수는 함수 객체를 가리기는 식별자로  호출

</aside>

### 함수 표현식

함수는 일급 객체이므로 함수 리터럴로 생성한 함수 객체를 변수에 할당 할 수 있다

```jsx
var add = function foo() {
	console.log('foo');
};
console.log(add(2,3)); //5
console.log(foo(2,3)); // ReferenceError 남
```

### Function 생성자 함수

생성자 함수에 매개변수 목록과 함수 몸체를 문자열로 전달하면 new 연산자와 함께 함수 객체를 생성하여 반환

```jsx
var add = new Function( ‘x’ + ‘y’, ‘return x + y’);
```

근데 이렇게 생성하는 일은 거의 없고 권장하지도 않으니 쓰지말자

클로저를 생성하지 않고 함수 선언문이나 표현식으로 생성한 함수와 다르게 동작한다

```jsx
var add = ( function () {
	var a = 10;
	return function (x , y) {
		return x+y+a;
		};
}()};

console.log(add(1,2)); //13

var add = ( function () {
	var a = 10;
	return  new Function( ‘x’ + ‘y’, ‘return x + y + a;’);
}()};
console.log(add(1,2)); //RefernceError : a is not defined
```

### 화살표함수

function 키워드 대신 화살표 ⇒ 를 이용하여 간랸하게 함수 선언하며 익명 함수로 정의한다

```jsx
var add = (x , y) ⇒ x + y ;
console.log(add(2,5));
```

**기존 함수와 조금 다른 점이 있다.** 

- 생성자 함수로 사용할 수 없음
- 기존 함수랑 this 바인딩 방시이 다름
- prototype 프로퍼티가 없음
- arguments 객체를 생성하지 않음

## 함수 호이스팅

함수 선언문이 코드 선두로 끌어 올려진 것 처럼 동작하는 자바스크립트 고유 특징

함수 선언문으로 정의한 함수는 함수 선언문 이전에 호출할 수 있으나 함수 표현식으로 정의한 함수는 호출할 수 없다

⇒ 함수 선언문과 함수 표현식의 생성 시점이 다르기 때문!
     함수 선언문은 런타임 이전에 실행되어 자바스크립트 엔진이 암묵적으로 식별자를 생성
     함수 표현식은 런타임에 평가 되므로 변수 호이스팅처럼 발생된다

> 하지만 변수 호이스팅과는 미묘한 차이가 있다. 변수는 undefined로 초기화 되고 암묵적 함수 식별자는 함수 객체로 초기화 된다.
> 

**함수 호이스팅은 함수를 호출 하기전에 반드시 함수를 호출해야 한다는 규칙을 무시하므로 함수 표현식 쓸 것을 권장한다** 

> ***2022-03-01 TIL***
함수는 값! 이므로 **식별자를 통해 구분**한다. ⇒ 함수 이름은 식별자인데 식별자 아님
**함수 표현식과 함수 선언식의 차이**
함수 선언식은 선언문이다. 그래서 자바스크립트가 암묵적으로 함수 이름과 같은 식별자를 생성한다. 
함수 표현식은 변수에 함수 객체를 할당한 것이다. 그래서 호이스팅에 영향받지 않는다 
그래서 선언하지도 않은 함수를 선언해도 실행되어 규칙을 무시하므로 함수 표현식을 쓰자
> 

## 함수 호출

함수는 함수를 가리키는 식별자와 함수 호출 연산자인 소괄호로 호출한다

함수를 호출하면 현재 실행 흐름을 중단하고 호출된 함수로 실행흐름을 옮긴다

매개변수에 인수가 순서대로 할당되고 함수 몸체의 문들이 실행된다

### 매개변수와 인수

함수 외부에서 함수 내부로 전달할 값이 있는 경우, 매개변수를 통해 인수를 전달

**인수**

값으로 평가 될 수 있는 표현식

함수를 호출할 때 지정, 개수와 타입에 제한 없음

**매개변수**

함수를 정의할 때 선언

함수 몸체 내부에서 변수와 동일하게 취급

함수가 호출되면 암묵적으로 몸체 내에 매개변수가 생성되고 undefined로 초기화 된 다음 인수가 할당된다

매개변수의 함수의 변수를 의미하고 인수는 함수에 넘겨주는 값

매개변수의 스코프는 함수 내부이다

### 인수 확인

자바스크립트는 매개변수와 인수의 개수가 일치하는지 확인하지 않고,
동적 타입 언어이기 때문에, 매개변수의 타입을 사전에 지정할 수 없다

그래서 함수에 어떤 타입의 인수를 전달해야 하는지, 어떤 타입의 값을 반환하는지 알 수 없기 때문에 학인할 필요가 있다

```jsx
function add(x, y){
	if(typeof x !== 'number' || typeof y !== 'number'){
		//매개변수 타입이 숫자가 아닌 경우 에러를 발생 시킴
		throw new TypeError('인자는 모두 숫자 값이여야 합니다.');
	}
	return x + y;
}
console.log(add(2)); //TypeError =>NaN
console.log(add('a','b')); //TypeError =>ab
```

**arguments 객체**를 통해 인수 개수를 확인 할 수 있다

**단축평가**를 사용해 매개변수에 기본값을 할당하는 방법도 있음

```jsx
function add(a, b, c){
	a = a || 0;
	b = b || 0;
	c = c || 0;
	return a + b + c;
}

console.log(add(1, 2, 3)); //6
console.log(add(1,2)); //3
console.log(add(1)); //1
console.log(add()); //0
```

**매개변수 기본값을 사용**하면 인수 체크 및 초기화를 간소활 할 수 있다

매개변수 기본값은 매개변수에 인수를 전달하지 않았을 경우, undefined인 경우에 유효

```jsx
function add(a = 0, b = 0, c = 0){
	a = a || 0;
	b = b || 0;
	c = c || 0;
	return a + b + c;
}
```

### 반환문

함수는 반환문을 사용해 실행 결과를 함수 외부로 반환할 수 있다

**역할**

- **함수의 실행을 중단**한다.  반환문 이후에 다른 문이 존재하면 무시된다.
- return 키워드 뒤에 오는 **표현식을 평가해 반환**한다. 표현식으로 지정하지 않으면 undefined가 반환된다

반환문이 생략되면 undefined를 반환한다

```jsx
function foo() {
}
console.log(foo()); //undefined
```

return 키워드와 표현식 사이에 줄바꿈이 있으면 사이에 세미콜론이 추가되어 오류가 발생할 수 있다.

```jsx
function foo() {
	return //줄바꿈이 되어 ;가 자동 삽입 된다
		'foo'; //무시됨
}
console.log(foo()); //undefined
```

## 참조에 의한 전달과 외부 상태의 변경

매개변수도 함수 몸체 내부에서 변수와 동일하게 취급되어 타입에 의해 전달 방식이 다르다

```jsx
function changeVal(primitive, obj) { //원시값, 객체
	primitive += 100;
	obj.name = 'kim';
}

var num = 100;
var person = {name : 'lee'};

console.log(num); //100
console.log(person); //{name : 'lee'}

changeVal(num, person);

console.log(num); //100  => 원시값은 원본이 훼손되지 않는다
console.log(person; //{name : 'kim'} =>객체는 원본이 훼손된다
```

<aside>
👉 **원시값은 값을 복사**해서 매개변수에 전달되기 때문에, **재할당**을 통한 교체를 해도 **원본은 훼손되지 않는다.**
**객체는 참조값이 복사**되어 매개변수에 전달되기 때문에, **참조값**을 통해 교체를 하면 **원본 값도 교체된다.**

**이런 문제는 불변 객체를 만들어 사용하는 것으로 해결한다!**

객체를 마치 원시값처럼 변경 불가능한 값으로 동작하게 만드는 것이다. 객체의 방어적 복사를 통해 원본 객체를 복제하여 새로운 객체를 생성하고 재할당을 통해 교체한다.

## 다양한 함수의 형태

### 즉시 실행함수

함수의 정의와 동시에 즉시 호출되는 함수, 그룹 연산자로 감싸서 표현한다

단 한번만 호출되며, 다시 호출할 수 없다

익명함수로 사용하는 것이 일반적이며 , 기명 즉시 실행함수로도 사용할 수 있다.

```jsx
==익명 즉시실행함수
(
	function(){
		return 'foo';
	}(); 
)
```

그룹 연산자 내의 기명함수는 리터럴로 평가되기 때문에 몸체에서만 참조할 수 있는 식별자 이므로 다시 호출할 수 없다

```jsx
==기명 즉시실행함수
(
	function foo(){
		return 'foo';
	}();
)

foo(); //ReerenceError
```

### 재귀 함수

재귀 호출을 수행하는 함수

재귀함수는 자신을 무한 재귀 호출하기 때문에 탈출 조건을 반드시 만들어야 한다

탈출 조건이 없으면 함수가 무한 호출되므로 스택 오버플로가 발생한다

```jsx
function factorial(n){
	if(n<=1) return 1; //탈출 조건
	return n * factorial(n-1);
}
```

### 중첩 함수

함수 내부에 정의된 함수를 중첩 함수 또는 내부 함수라 한다.

중첩함수는 자신을 포함하는 외부 함수를 돕는 헬퍼 함수의 역할을 한다

```jsx
function foo() {//외부 함수

		function poo() { //내부함수
			return 'poo';
		}
		poo();
		return 'foo';
	}

foo();
```

### 콜백함수

함수의 매개변수를 통해 다른 함수의 내부로 전달되는 함수

**고차함수와 콜백함수의 관계**

매개변수를 통해 콜백함수를 전달받은 함수를 고차함수라고 한다.

고차함수는 콜백함수를 자신의 일부분 처럼 합성하여 콜백 함수의 호출 시점을 결정해서 호출한다.

**콜백함수는 고차 함수에 의해 호출되고, 고차함수는 필요에 따라 콜백 함수에 인수를 전달할 수 있다**

콜백함수로 전달된 함수 리터럴은 고차함수가 호출할 때마다 평가되어 객체를 생성한다.

**⇒ 콜백 함수를 다른 곳에서도 호출할 필요가 있거나 함수가 자주 호출된다면 고차함수에 정의하는 것이 효율적이다.**

```jsx
function repeat( n ,f ){
	for (var i = 0; i < n ; i++){
		f(i);
	}
}

var logAll = function (i) {
	console.log(i);
}
//반복 호출할 함수를 인수로 전달한다
repeat(5, logAll); //0 1 2 3 4 

var logOdds = functon (i) {
	if ( i % 2 ) console.log(i);
};

repeat(5, logOdds); // 1 3
	
```

### 순수함수와 비순수 함수

**순수함수 :** 부수 효과가 없는 함수

**비순수함수 :** 외부상태에 의존하거나 외부 상태를 변경하는(=부수효과) 함수

```jsx
var count = 0;
function increase() {
	return ++count;
}
increase();
console.log(count); //1

increase();
console.log(count); //2

```

위의 예제와 같이 인수를 전달받지 않고 함수 내부에서 외부 상태를 직접 참고하면 외부 상태에 의존하게 된다.  함수가 외부 상태를 변경하면 상태 변화를 추적하기 어려워 진다. 

**따라서 함수 외부 상태를 변경을 지양하는 순수 함수를 사용하는 것이 좋다.**

함수형 프로그래밍은 조건문과 반복문을 제거해서 복잡성을 해결하고, 변수 사용을 억제하거나 생명주기를 최소화해서 오류를 최소화하는 것을 목표로 한다. 부수 효과를 억제하는 것은 오류를 피하고 프로그램의 안전성을 높이려는 노력의 일환이므로 순수 함수를 사용하자!
