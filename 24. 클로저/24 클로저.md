## 24. 1 렉시컬 스코프

> 클로저는 함수와 그 함수가 선언된 렉시컬 환경과의 조합이다
> 

```jsx
const x = 1;

funciton outerFunc(){
	const x = 10;

function innerFunc(){
	console.log(x);
	}
	innerFunc();
}

outerFunc();
```

**→ 함수 innerFunc가 outerFunc 함수의 내부에 정의된 중첩함수가 아니라면 innerFunc 함수를 outerFunc 함수의 내부에서 호출해도 outerFunc 함수의 변수에 접근 할 수 없다**

**→ 이러한 이유는 자바스크립트가 렉시컬 스코프를 따르는 언어이기 때문이다**

> 자바스크립트 엔진은 함수를 어디서 호출했는지가 아니라 어디서 정의했는지에 따라 상위스코프를 결정한다. 이를 렉시컬 스코프라고 한다.
> 

## 24. 2 함수 객체의 내부 슬롯 [[Environment]]

- 렉시컬 스코프가 가능하려면 함수는 자신이 정의된 환경(함수 정의가 위치하는 스코프가 상위스코프)을 기억해야한다.
- 함수는 자신의 내부 슬롯 [[Environment]]에 자신이 정의된 환경을 저장한다.

## 24. 3 클로저와 렉시컬 환경

```jsx
const x = 1;

funciton outer(){
	const x = 10;
	const inner = function(){console.log(x);};
	return inner;
}

const innerFunc = outer();
innerFunc();  // 10
```

→ 위의 예시에서 outer 함수가 호출되고 inner가 리턴되고 나면 outer는 생명주기를 마감하게 된다. 그릐고 실행컨텍스트 스택에서도 제거된다. 그렇기 때문에 outer의 지역변수도 더이상 유효하지 않아 ‘10’이라는 값이 나올 것 같지 않지만 결과는 그렇지 않다.

**이처럼 외부함수보다 중첩함수가 더 오래 유지되는 경우 중첩함수는 이미 생명주기가 종료한 외부함수의 변수를 참조할 수 있다. 이를 클로저라고 부른다.**

가비지 컬렉터는 참조하고 있는 메모리공간을 함부로 해제하지 않기 때문에 외부함수의 생존과 관계 없이 내부함수에서 상위 스코프의 식별자를 참조 할 수 있고 식별자의 값을 변경 할 수도 있다.

## 24. 4 클로저의 활용

```jsx
const increaseType1 = function(){
    let num = 0;
    return ++num;
};

const increaseType2 = (function(){
    let num = 0;
    // 클로저
    return function(){
        return ++num;
    }
}());

console.log(increaseType1());  // 1
console.log(increaseType1());  // 1

console.log(increaseType2());  // 1
console.log(increaseType2());  // 2
```

→ 클로저는 상태가 의도치 않게 변경되지 않도록 은닉하고 특정 함수에게만 상태변경을 허용하여 상태를 안전하게 변경하고 유지하기 위해 사용한다.

### 24. 5 캡슐화와 정보 은닉

> 캡슐화란 객체의 상태를 나타내는 프로퍼티와 프로퍼티를 참조하고 조작 할 수 있는 동작인 메서드를 하나로 묶는 것을 말한다. 캡슐화는 객체의 특정 프로퍼티나 메서드를 감출 목적으로 사용하기도 하는데 이를 정보은닉이라 한다.
> 

- 정보 은닉은 객체간의 상호 의존성(결합도)를 낮추는 효과가 있다.
- 자바스크립트에서 객체의 모든 프로퍼티와 메서드는 기본적으로 public 하다

### 24. 6 자주 발생하는 실수

```jsx
var funcs = [];

for(var i = 0; i < 3; i++){
    funcs[i] = function(){ return i; };
}

for(var j = 0; j < funcs.length; j++){
    console.log(funcs[j]());
}
// -> i가 전역 변수이기 때문에 0, 1, 2의 값이 할당되고 func의 배열 요소 함수를 호출하면
// 이미 for문을 돌고 할당 돼있는 값 3이 출력된다. 

// ------------------------------------------------------------------------------------

var funcs = [];

for(var i = 0; i < 3; i++){
	funcs[i] = (function (id){
		return function(){
			return id;
		};
	}(i));
}

for(var j = 0; j < funcs.length; j++){
	console.log(funcs[j]());
}
```
